# Speaker Notes Strategy

> Plan the notes approach here before writing slide-by-slide notes.
> Slide-by-slide notes live alongside the slide content in the content files.

---

## Notes Function

Speaker notes for this presentation should serve as:

- [ ] Delivery reminders for the speaker (key triggers, not a script)
- [ ] Standalone document for async viewers
- [ ] Transcript for recorded version
- [ ] Both delivery reminders AND standalone document

*(Standalone / transcript mode requires complete prose — every key verbal point must appear in notes, not just on slides.)*

## Transition Strategy

Every slide should have an explicit verbal transition in the notes.

**Transition style preference:**
- [ ] Logical connectors ("and this is why…")
- [ ] Question pivots ("so what does that mean for…")
- [ ] Contrast pivots ("but here's what we didn't expect…")
- [ ] Mix — write what fits each moment
