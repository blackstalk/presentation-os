# Examples

Reference implementations of complete presentations built using Presentation OS.

---

## Purpose

Examples serve as:
- Proof that the platform works end-to-end
- Reference implementations for patterns documented in `docs/`
- Learning resources for future presentations

---

## What Makes a Good Example

A good example:
- Represents a complete presentation (brief through delivery)
- Demonstrates non-obvious patterns that benefit from a reference
- Includes a retrospective note capturing what worked and what to carry forward
- Is documented well enough that someone unfamiliar with the project can learn from it

---

## Adding an Example

When adding an example:

1. Create a directory: `examples/[presentation-slug]/`
2. Include the project `CLAUDE.md`, `brief.md`, `story-map.md`, and final output
3. Add a `retro.md` with retrospective notes
4. Add an entry to the registry below
