# Components

Reusable presentation components — patterns that recur across presentations and benefit from a standardized implementation.

---

## What Belongs Here

A component is a reusable pattern that:
- Appears in more than one presentation type
- Has a standard implementation that should be consistent
- Benefits from documentation about when and how to use it

---

## Adding a Component

When adding a component:

1. Create a directory: `components/[component-name]/`
2. Include implementation file(s) and a `README.md` with usage guidance
3. Document: when to use, when not to use, accessibility notes, customization options
4. Add an entry to the registry below
